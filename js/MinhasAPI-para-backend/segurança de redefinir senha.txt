Limitar tentativas de uso de token (por IP / por e-mail).

Honeypot fields e reCAPTCHA no forgot-password para reduzir abuse.

Verificação de password strength (zxcvbn) no frontend e backend.

Bloquear token se usuário mudou a senha entre a requisição e uso (no exemplo, usamos used + substituímos token na próxima req).

Logs e alertas: e-mail para usuário quando a senha é alterada e quando há várias tentativas falhas.

Política de expiração: 1 hora é comum, 15–30 min mais seguro.

Lista de senhas proibidas (123456, etc.).

HSTS + CSP no site.

Proteção CSRF em formulários (para endpoints que não são pure API).

HTTP-only cookies para sessão; mas token de reset é enviado por link (query param) — alguns preferem usar token no path ou em body via POST após abrir página; se usar query param, trate de limpar o URL do navegador após uso (replaceState).
